一、Markdown基本语法

1. 标题

Markdown支持两种标题的语法，类Setext和类atx形式。类Setext形式是用底线的形式，利用=（最高阶标题）和-（第二阶标题），例如：

This is an H1
=======

This is an H2
-————
显示效果：

This is an H1

This is an H2

任何数量的=和-都可以有效果。

需要注意一点，由于分割线也是‘——’，因此在使用分割线时，一定要空一行，不然会把上方文字识别为第二阶标题。原因会在后面的段落和换行种说到。

类Atx形式则是在行首插入1到6个#号，对应标题1到6阶，示例：

# this is H1

## this is H2

### this is H3

#### this is H4

##### this is H5

###### this is H6
显示效果：

this is H1

this is H2

this is H3

this is H4

this is H5
this is H6

要注意的是，#号后面必须跟一个空格再写文本，不然无法识别为标题。
按规范来说，同一份文件中一阶标题只能有一个，且标题前后都要空一行。推荐使用#号的方式。

2. 字体

Markdown使用星号*、底线_还有波浪~（英文输入法下）作为标记强调字词的符号，用单星号或单底线括起来的文字会倾斜，用双星号或双底线括起来的文字会加粗，用三星号或三底线括起来的文字会倾斜并加粗，用双波浪号括起来的文字会加删除线。

你可以随便用你喜欢的样式，唯一的限制是，你用什么符号开启标签，就要用什么符号结束。个人感觉写中文时还是使用*号比较方便，因为它不区分全角半角，不用切换输入法。示例：

**这是加粗**
__这也是加粗__
*这是倾斜*
_这也是倾斜_
***这是加粗倾斜***
___这也是加粗倾斜___
~~这是加删除线~~

效果如下：
这是加粗
这也是加粗
这是倾斜
这也是倾斜
这是加粗倾斜
这也是加粗倾斜
这是加删除线

推荐使用星号。

注意：上述效果可是直接插在文字中间，如果你想在文字前后直接插入普通的*、_或~,可以使用\，这样他们就只会被当成普通的符号。比如这样： *哈基米* 。

3. 分割线

你可以在一行中用三个以上的*、-或_来建立一个分隔行，行内不能有其他东西。你也可以在星号或是减号中间插入空格。下面每种写法都可以建立分隔线：
* * *
***
************
- - -
-–
-——–
_ _ _
___
_________

效果展示：

推荐使用* * *的方式。

4. 引用

在引用的文字前加>即可。示例：

This is a blockquote with tow paragraphs. Lorem ipsum dolor sit amet, consectetuer adipiscing elit. Aliquam hendrer it mi posuere lectus. Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus. Donec sit amet nisl. Aliquam semper ipsum sit amet velit. Suspendisse id sem consectetuer libero luctus adipiscing.te with two paragraphs.

Markdown也允许你只在整个段落的第一行最前面加上>:

This is a blockquote with two paragraphs. Loram ipsum dolor sit amet, consectrtuer adipiscing elit. Aliquam hendrerit mi posuere lectus. Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus. Donec sit amet nisl. Aliquam semper ipsum sit amet velit. Suspendisseid sem consectetuer libero luctus adipiscing.

引用文本和其他文本之间需要用空行隔开。
区块引用可以嵌套（引用中的引用），只要根据层次加上不同数量的>:

This is the first lecel of quoting.

This is nested blockquote.

Back to the first level.

引用的区块内也可以使用其他的Markdown语法，包括标题、列表、代码区块等。

5. 列表

Markdown支持有序列表和无序列表。无序列表使用星号、加号或者减号作为列表标记。例如：

- 列表内容
+ 列表内容
* 列表内容
效果展示：

  • 列表内容
  • 列表内容
  • 列表内容

注意：-、+、*号后面都要有一个空格。推荐用-。
有序列表则使用数字接着一个英文句点作为标记。示例：

1. 列表内容  
2. 列表内容  
3. 列表内容

效果展示：

  1. 列表内容
  2. 列表内容
  3. 列表内容

注意：.后面要有一个空格。
还有很重要的一点，你在列表标记上使用的数字只有第一个会影响最终显示结果。 示例：

1. hajimi
1. mijiha
1. hamiji

效果展示：

  1. hajimi
  2. mijiha
  3. hamiji

或者

8. mihaji
1. jihami
4. mihaji

效果展示：

  1. mihaji
  2. jihami
  3. mihaji

两段列表标记之间必须用空行加文本隔开，否则会被归类为同一块列表，按最前面的列表序号依次排序

列表可以嵌套，上一级和下一级之间敲一个tab键或几个空格即可。

  • 一级无序列表内容
    • 二级无序列表内容
    • 二级无序列表内容

要让列表看起来更漂亮，你可以把内容用固定的缩进整理好：

  • Lorem ipsum dolor sit amet, consectetuer adipiscing elit. Aliquam hendrerit mi posuere lectus. Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus.
  • Donec sit amet nisl. Aliquam semper ipsum sit amet velit. Suspendisse id sem consectetuer libero luctus adipiscing.

6. 表格

示例：

表头|表头|表头
:--|:--:|--:
左对齐|居中|右对齐
内容|内容|内容

效果展示：

表头 表头 表头
左对齐 居中 右对齐
内容 内容 内容

第二行分割表头和内容，-符号有一个就行，为了对齐，多加了几个。
文字默认居左,或者在-左边加:
-符号两边加:表示文字居中
-符号右边加:表示文字居右

7. 代码

在Markdown中加入代码块有两种方式：第一种，只要简单地缩进4个空格或是一个制表符就可以，但是必须与前面的文本用空行隔开

这是一个普通段落：

这是一个代码区块

第二种方法更为常用，

单行代码：代码之间分别用一个反引号包起来即可：
这里有一句代码代码内容

代码块：代码之间分别用三个反引号包起来，且两边的反引号单独占一行，这种方式支持显示语法高亮显示，但是必须指明使用的编程语言。这种方式比较现代，推荐使用这种方式。

```python
print('hello')
```

```c
#include <iostream>

int main()
{
  printf("hello");
  return 0;
}
```

效果展示：

print('hello')

#include <iostream>

int main()
{
  printf("hello");
  return 0;
}

8. 段落和换行

一个Markdown段落是由一个或多个连续的文本行组成，他的前后要有一个以上的空行（空行的定义是显示上看起来是像是空的，便会被视为空行。比方说，若一行只包含空格和制表符，则改行就会被视为空行）。普通段落不该用空格或制表符来缩进。

我们在两个不同的文字块之间，一定要空行以示区分，不然就会被归入同一文字块中。Markdown允许段落内的强迫换行（插入两个空格+换行符）。
